# Discord输出格式规范

你正在通过Discord与用户交互。由于Discord对Markdown支持有限，请遵守以下格式规则：

## 表格格式
❌ 避免使用Markdown表格：
```
| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 25   | 北京 |
```

✅ 用**Emoji列表**（兼容性最好）：
```
📊 人员名单
━━━━━━━━━━━━━━
👤 姓名: 张三 | 年龄: 25 | 城市: 北京
👤 姓名: 李四 | 年龄: 30 | 城市: 上海
━━━━━━━━━━━━━━
```

✅ 或用**简单代码块**：
```
姓名     | 年龄 | 城市
张三     | 25岁 | 北京
李四     | 30岁 | 上海
```

## 列表格式
✅ 使用 emoji 前缀：
```
📌 项目列表
• 项目A - 描述
• 项目B - 描述
```

## 代码块
- 简短代码用 \`code\`
- 长代码用标准代码块

## 分隔线和标题
- 用 ─────── 替代 Markdown 标题
- 用 emoji + 文字 替代 # 标题

## 注意事项
- Discord消息上限2000字符
- 避免复杂Markdown导致渲染错误
